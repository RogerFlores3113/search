# Milestone v0.1.0: MVP

**Status:** ✅ SHIPPED 2026-05-16
**Phases:** 1-4
**Total Plans:** 10

## Overview

Four phases from clean scaffold to distributable Mac app. Phase 1 proved the agentic loop works from the CLI with one model. Phase 2 added all model providers and hardened guardrails. Phase 3 wrapped everything in a full localhost web UI — prompt box, live screenshot stream, narration feed, pause/stop. Phase 4 packaged it as a double-click Mac .app with CI/CD.

The apartment search was a v1 *test case* to validate the loop, not a hardcoded preset.

## Milestone Summary

**Key Decisions:**

- browser-use 0.12.6 as loop engine (Stagehand is TypeScript-only; raw Playwright would require months of custom agentic loop work)
- litellm>=1.83.0 hard pin (supply chain backdoor in 1.82.7/1.82.8 — neutralized from day 1)
- FastAPI + HTMX + SSE for UI (zero build step, 30-40KB JS vs React's 100-300KB)
- asyncio.Queue as agent-to-SSE bridge (decouples agent from HTTP layer; single event loop)
- v1 is general-purpose loop (any site, any task) — presets are v2
- Training data collection (screenshots + actions) is v1 — LoRA pipeline is v2
- PyInstaller for distribution (uv/pyproject approach was considered but bundled .app is the consumer target)
- macOS only for v1 (Linux/Windows CI deferred — cdp-use replaces Playwright so macOS bundle is the validated path)

**Issues Resolved:**

- Chrome v136 CDP workaround — resolved via `channel="chrome"` and cdp-use in Phase 4
- `Browser(keep_alive=True)` hang — resolved by using BrowserSession(keep_alive=False) from Phase 1
- litellm removed from browser-use 0.12.5 core — resolved by installing litellm separately, using ChatLiteLLM for API providers
- Sync Playwright in asyncio crash — resolved via `async_playwright()` pattern

**Issues Deferred:**

- Manual smoke test (01-03 Task 2) — requires human with Chrome + Ollama
- macOS bundle codesign/notarize — ad-hoc signing ships; full Apple notarization is v2
- Windows .exe / Linux AppImage — CI scaffolded but macOS is the validated distribution path
- Phase 01-03 01-MANUAL-VERIFICATION.md — pending human checkpoint

**Technical Debt Incurred:**

- ROADMAP progress table was not updated during execution (fixed at milestone close)
- REQUIREMENTS.md traceability not updated after Phase 1 (fixed at milestone close)
