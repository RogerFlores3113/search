# Requirements Archive: v0.1.0 — local-browser-agent

**Archived:** 2026-05-16
**Defined:** 2026-05-13
**Core Value:** User types any natural language task, the agent opens Chrome and completes it — proving the general-purpose agentic loop works end-to-end before any structured presets are built.

> This is the requirements snapshot at v0.1.0 milestone close. All v1 requirements were implemented across Phases 1-4.
> For current requirements, see the fresh REQUIREMENTS.md created for the next milestone.

---

## v1 Requirements — ALL COMPLETE ✅

### Agent Loop

- [x] **LOOP-01**: User enters a natural language task in the UI prompt box and submits it to start a run — *implemented in Phase 3 (03-01)*
- [x] **LOOP-02**: Agent runs a screenshot → LLM decision → browser action cycle via browser-use until the task is complete or a limit is reached — *implemented in Phase 1 (01-02)*
- [x] **LOOP-03**: Agent drives the user's installed Chrome browser (headed, visible, residential IP) for all browsing — *implemented in Phase 1 (01-02)*
- [x] **LOOP-04**: LLM has full browser control — click any element, type into any field, scroll, navigate to any URL — *implemented in Phase 1 (01-02)*
- [x] **LOOP-05**: Agent terminates automatically after a configurable `max_steps` limit (default: 25) to prevent infinite loops — *implemented in Phase 1 (01-02)*
- [x] **LOOP-06**: Agent terminates automatically after a configurable session timeout (default: 10 minutes) — *implemented in Phase 1 (01-02)*
- [x] **LOOP-07**: User can pause the agent mid-run from the UI — *implemented in Phase 3 (03-03)*
- [x] **LOOP-08**: User can stop the agent mid-run; Chrome closes cleanly with no orphan processes — *implemented in Phase 3 (03-03)*
- [x] **LOOP-09**: Each agent step emits a plain-language narration event streamed live to the UI — *implemented in Phase 3 (03-01)*
- [x] **LOOP-10**: Agent returns a plain-language summary of what it accomplished at the end of each run — *implemented in Phase 3 (03-02)*

### Model Support

- [x] **MODEL-01**: User can select Ollama as the AI provider; any installed Ollama vision model is accepted; Qwen2.5-VL:7b is the recommended default shown in the UI — *implemented in Phase 1 (01-02)*
- [x] **MODEL-02**: User can select Anthropic (Claude) as the AI provider by entering an API key; key is stored locally only (SecretStr), never transmitted anywhere except Anthropic's API — *implemented in Phase 2 (02-01)*
- [x] **MODEL-03**: User can select OpenAI (GPT-4o) as the AI provider by entering an API key; key is stored locally only (SecretStr), never transmitted anywhere except OpenAI's API — *implemented in Phase 2 (02-01)*
- [x] **MODEL-04**: App validates the selected model is reachable before starting a run; shows a clear, actionable error message if not — *implemented in Phase 1 (01-02)*

### Guardrails

- [x] **GUARD-01**: A configurable global domain blocklist prevents the agent from loading pages on restricted domains; blocked navigations are logged and surfaced to the user — *implemented in Phase 2 (02-02) via BrowserProfile.prohibited_domains + SecurityWatchdog*
- [x] **GUARD-02**: Agent is instructed via system prompt not to click "Buy Now", "Purchase", "Checkout", "Pay", or equivalent CTAs — *implemented in Phase 2 (02-02) via extend_system_message=GUARDRAIL_PROMPT*
- [x] **GUARD-03**: Agent is instructed not to submit credentials, payment info, or personal data on any site the user did not explicitly name — *implemented in Phase 2 (02-02) via GUARDRAIL_PROMPT*
- [x] **GUARD-04**: If a CAPTCHA is detected, the agent pauses and notifies the user in the UI; the run does not fail silently — *implemented in Phase 2 (02-02) via log_step CAPTCHA keyword detection*

### Live UI

- [x] **UI-01**: User enters a natural language task in a prompt box and presses Enter (or clicks Run) to start the agent — *implemented in Phase 3 (03-01)*
- [x] **UI-02**: The agent's current browser viewport is captured at each step and streamed to the UI — *implemented in Phase 3 (03-02) via ScreenshotEvent*
- [x] **UI-03**: A live narration feed shows a plain-language description of each action as it happens, updating in real time via SSE — *implemented in Phase 3 (03-01)*
- [x] **UI-04**: UI shows the current agent state at all times: idle / running / paused / complete / error — *implemented in Phase 3 (03-01)*
- [x] **UI-05**: UI shows a progress indicator: steps taken vs. max steps, and elapsed time — both updating live — *implemented in Phase 3 (03-02) via ProgressEvent + Alpine counter*
- [x] **UI-06**: Pause button halts the agent mid-step; Resume continues from where it stopped — *implemented in Phase 3 (03-03)*
- [x] **UI-07**: Stop button terminates the agent immediately and closes the browser cleanly — *implemented in Phase 3 (03-03)*
- [x] **UI-08**: Failed runs display a plain-language error message — not a Python stack trace — *implemented in Phase 3 (03-02)*
- [x] **UI-09**: Completed runs display the agent's plain-language summary of what it accomplished — *implemented in Phase 3 (03-02) via SummaryEvent*

### Run History & Training Data

- [x] **RUN-01**: Each run's task prompt, narration log, final summary, and status are saved locally on completion — *implemented in Phase 3 (03-03) via aiosqlite insert_run*
- [x] **RUN-02**: User can view a list of recent runs showing: task prompt (truncated), status, and timestamp — *implemented in Phase 3 (03-03) via /runs endpoint + history panel*
- [x] **RUN-03**: Every agent step is logged as a training record: `{screenshot_b64, action_type, action_target, action_value, narration, step_success}` saved to a local JSONL file — *implemented in Phase 1 (01-02) via log_step*

### Distribution

- [x] **DIST-01**: User can launch the app on Mac by double-clicking a `.app` bundle — no Python, no terminal, no SDK required — *implemented in Phase 4 (04-01, 04-02)*
- [x] **DIST-02**: App detects the user's installed Google Chrome and drives it via `channel="chrome"`; if Chrome is not found, a friendly prompt explains what to install — *implemented in Phase 4 (04-01) via chrome_is_installed() + no_chrome.html*
- [x] **DIST-03**: App opens the user's default browser to `localhost:8080` automatically within 3 seconds of launch — *implemented in Phase 4 (04-01) via schedule_browser_open() threading.Timer*
- [x] **DIST-04**: First-time users see a one-time safety disclaimer before any run starts — *implemented in Phase 4 (04-01) via Alpine.js + localStorage gate*
- [x] **DIST-05**: GitHub Actions pipeline builds and publishes the `.app` to GitHub Releases automatically on each version tag push — *implemented in Phase 4 (04-02) via .github/workflows/release.yml*
